// Limpieza de outliers mediante Rango Intercuartilico (IQR)
function filterOutliers(data, key) {
  const values = data.map(d => d[key]).sort((a, b) => a - b);
  const q1 = values[Math.floor(values.length * 0.25)];
  const q3 = values[Math.floor(values.length * 0.75)];
  const iqr = q3 - q1;
  const lowerBound = q1 - 1.5 * iqr;
  const upperBound = q3 + 1.5 * iqr;
  
  return data.filter(d => d[key] >= lowerBound && d[key] <= upperBound);
}
\end{lstlisting}

\subsection{Objetivo Específico 3: Implementación del Algoritmo MCDA}
El algoritmo de decisión multicriterio realiza la normalización de datos e introduce la penalización dura para garantizar la exclusión de plugins no seguros.

\begin{itemize}
    \item \textbf{Archivo Clave:} \path{cni-recommender-spa/src/data/recommendationModel.js}
    \item \textbf{Función de Normalización y Scoring MCDA:}
\begin{lstlisting}
// Calculo del ranking MCDA con penalizacion no compensatoria
export function calculateScores(profile, benchmarkPayload) {
  const cnis = benchmarkPayload ? buildCniMetricsFromBenchmark(benchmarkPayload) : cniMetrics;
  const securityNeed = profile?.securityNeed ?? 0;
  const securityIsRequired = securityNeed >= 4; // Umbral de restriccion

  return cnis.map((cni) => {
    const rawTotal = Object.entries(profile.weights).reduce((sum, [metric, weight]) => {
      const metricValue = cni.scores[metric] ?? 0;
      return sum + metricValue * weight;
    }, 0);

    // Si requiere seguridad y el CNI no la soporta, penalizar 60%
    const securityPenalized = securityIsRequired && cni.supportsNetworkPolicy === false;
    const finalScore = securityPenalized ? rawTotal * 0.4 : rawTotal;

    return {
      ...cni,
      score: Number(finalScore.toFixed(3)),
      rawScore: Number(rawTotal.toFixed(3)),
      securityPenalized,
    };
  }).sort((a, b) => b.score - a.score);
}
\end{lstlisting}
\end{itemize}

\section{Guía de Reproducción y Verificación Científica} \label{sec:apa_reproducibilidad}

Para compilar, validar y reproducir el entorno y el recomendador de forma local, siga la secuencia de comandos detallada a continuación.

\subsection{Paso 1: Clonación del Repositorio y Reconstrucción de Datos}
Clonar el repositorio principal del proyecto y ejecutar el procesador automatizado de Node.js:
\begin{lstlisting}[language=bash]
# Clonar el proyecto
git clone https://github.com/DuwanSierra/K8S-CNI-Results.git
cd K8S-CNI-Results

# Instalar dependencias del pipeline y procesar datos brutos
node docs/procesador.js
\end{lstlisting}
Este comando procesará los datos brutos guardados bajo \path{results/cni-benchmarks/}, calculará la limpieza IQR de outliers y generará el archivo unificado en \path{results/processed/thesis_data.json}.

\subsection{Paso 2: Compilación y Despliegue de la SPA Recomendadora}
Navegar al directorio de la aplicación web, instalar dependencias de desarrollo local y levantar el servidor web:
\begin{lstlisting}[language=bash]
# Cambiar al directorio del recomendador web
cd cni-recommender-spa

# Instalar dependencias de React
npm install

# Lanzar el servidor de desarrollo local
npm run dev
\end{lstlisting}
El servidor local se desplegará en \url{http://localhost:5173/}, permitiendo interactuar con los deslizadores y replicar los perfiles Fintech, Streaming e IoT descritos en la investigación.
