\chapter{Recomendaciones} \label{sec:Recomendaciones}

Las siguientes recomendaciones se derivan de los hallazgos del experimento y están dirigidas a equipos que evalúen o desplieguen clústeres Kubernetes en producción:

\begin{itemize}
    \item \textbf{Restringir el uso de Flannel:} Se recomienda evitar el despliegue de Flannel en cualquier clúster de Kubernetes en producción debido a su incapacidad para aplicar Network Policies, limitando su uso únicamente a entornos de pruebas o desarrollo local de baja complejidad.
    
    \item \textbf{Seleccionar Cilium para seguridad crítica L7:} Para organizaciones que implementen microservicios transaccionales bajo arquitectura Zero Trust (perfil Fintech) y requieran control a nivel de API HTTP (capa 7), se recomienda adoptar Cilium para aprovechar el filtrado eBPF y la microsegmentación dinámica. En perfiles de Streaming o alto rendimiento sin necesidad de políticas complejas, se aconseja Flannel.
    
    \item \textbf{Adoptar Flannel en entornos de recursos limitados (Edge/IoT):} Debido a que Calico y Cilium demandan una huella de memoria RAM y CPU sustancialmente mayor ($1672.83$ MB y $1497.19$ MB respectivamente bajo estrés, frente a los $1176.54$ MB y $19.45\%$ CPU de Flannel), se aconseja emplear Flannel en clústeres operados sobre hardware limitado, siempre que la seguridad perimetral del clúster sea suficiente y no se requieran Network Policies.
    
    \item \textbf{Mantener la configuración de red y seguridad vía GitOps:} Se recomienda automatizar y versionar tanto el despliegue de los plugins CNI como la declaración de las Network Policies en un repositorio de Git acoplado a ArgoCD, asegurando la restauración automática de la configuración ante desvíos manuales no autorizados.

    \item \textbf{Tratar el modelo MCDA como un proceso de decisión continuo:} El modelo de decisión multicriterio no debe entenderse como una evaluación puntual, sino como un proceso periódico. A medida que la organización incorpore nuevas cargas de trabajo, actualice las versiones de los CNIs o cambie sus prioridades de negocio, recalibrar las métricas y los pesos asegura que la selección de infraestructura refleje el estado real del entorno y no una decisión congelada en el tiempo.
\end{itemize}
