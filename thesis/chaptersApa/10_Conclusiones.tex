\chapter{Conclusiones} \label{sec:Conclusiones}

Con base en el desarrollo experimental y los resultados obtenidos en este trabajo de grado, se presentan las siguientes conclusiones organizadas por cada uno de los objetivos específicos:

\begin{itemize}
    \item \textbf{Evaluación cuantitativa de rendimiento (OE1):} Las pruebas controladas demostraron que el dataplane de Cilium, impulsado por eBPF, ofrece la mayor eficiencia de red con un throughput promedio de 9.64 Gbps y la menor latencia de conexión (0.312 ms). Alternativas de VXLAN tradicionales como Flannel e implementaciones sobre switch virtual como Antrea mostraron latencias superiores y una mayor tasa de retransmisión de paquetes.
    
    \item \textbf{Políticas de seguridad Zero Trust (OE2):} Se comprobó que Calico, Cilium y Antrea realizan el enforcement correcto de las Network Policies con un overhead mínimo en el rendimiento (adición de latencia de entre 1.9\% y 5.1\%). De forma crítica, se determinó que Flannel carece de plano de control de seguridad en su plano de datos, por lo que no hace efectivas las políticas aplicadas, representando un riesgo de seguridad latente y silencioso para el clúster.
    
    \item \textbf{Desarrollo del modelo MCDA (OE3):} El modelo de decisión multicriterio desarrollado demostró ser una herramienta robusta para reconciliar requerimientos contrapuestos de QoS, recursos y seguridad. La normalización min-max y la limpieza estadística mediante el rango intercuartílico (IQR) permitieron evaluar con objetividad a los diferentes CNIs según el perfil empresarial (seguridad para Fintech, throughput para Streaming y bajo consumo de memoria RAM para IoT).
    
    \item \textbf{Prototipo recomendador web (OE4):} El desarrollo de la aplicación web SPA basada en React e integrada a un pipeline automatizado validó la factibilidad de exponer los resultados del modelo MCDA de forma interactiva y amigable. Las pruebas funcionales confirmaron la consistencia e integridad de los datos end-to-end sin pérdida de precisión.
\end{itemize}
